# Phase 4 — Networking

Containers communicate over networks. Master Docker's network drivers, DNS, load balancing, and troubleshooting.

---

## Topics

1. **Network drivers** — bridge, host, none, overlay, macvlan, ipvlan
2. **Default vs user-defined bridge** — why user-defined is better
3. **Embedded DNS & service discovery** — how containers find each other
4. **Port publishing** — EXPOSE, port mapping, localhost vs network
5. **Overlay & VXLAN** — multi-host networking (Swarm)
6. **Ingress & routing mesh** — service load balancing
7. **CNI overview** — why Kubernetes uses CNI
8. **Load balancing** — L4 vs L7, how Docker handles it
9. **Reverse proxies** — Nginx, Traefik, Caddy, HAProxy
10. **Network troubleshooting** — nsenter, tcpdump, ss, iptables inspection

---

## Learning objectives

✅ Can create and connect containers to networks  
✅ Understand bridge driver vs overlay  
✅ Can debug DNS resolution between containers  
✅ Can configure port publishing  
✅ Understand Swarm routing mesh  
✅ Can troubleshoot network issues with tcpdump  
✅ Can design multi-host networking  
✅ Can setup reverse proxies in containers
