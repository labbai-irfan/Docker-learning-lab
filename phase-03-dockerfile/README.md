# Phase 3 — Dockerfile Mastery

Optimize images for size, security, build time, and reproducibility.

---

## Topics

1. **All instructions** — FROM, RUN, CMD, ENTRYPOINT, COPY, ADD, ENV, ARG, WORKDIR, EXPOSE, LABEL, USER, VOLUME, HEALTHCHECK, SHELL, STOPSIGNAL, ONBUILD
2. **Layer mechanics & cache** — understanding build cache, cache invalidation
3. **Multi-stage builds** — separate build stages, size optimization
4. **BuildKit features** — cache mounts, secrets, SSH, heredocs
5. **.dockerignore** — what to exclude from build context
6. **ARG vs ENV, CMD vs ENTRYPOINT** — when to use each, exec vs shell form
7. **Optimization & reproducible builds** — size reduction, deterministic builds

---

## Learning objectives

✅ Can write optimized Dockerfiles from scratch  
✅ Understand layer caching and when it breaks  
✅ Multi-stage builds for size reduction  
✅ Can explain CMD vs ENTRYPOINT (exec vs shell)  
✅ Can use BuildKit secrets, cache mounts, SSH  
✅ Can debug Dockerfile build failures  
✅ Can design distroless/scratch-based images  
✅ Can write reproducible, deterministic builds
