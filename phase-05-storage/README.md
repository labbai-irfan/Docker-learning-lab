# Phase 5 — Storage & Data

Containers are ephemeral, but data must persist. Volumes, bind mounts, storage drivers, distributed storage.

---

## Topics

1. **Volumes** — named/anonymous, volume drivers
2. **Bind mounts** — mount host paths, use cases
3. **tmpfs mounts** — in-memory storage
4. **Storage drivers & backends** — overlay2, AUFS, ZFS, btrfs
5. **Network storage** — NFS, CIFS/SMB, EFS, EBS
6. **Distributed storage** — Ceph, GlusterFS, Longhorn, Portworx
7. **Backup & restore** — strategies for stateful containers
8. **Stateful workloads** — designing persistent apps in containers

---

## Learning objectives

✅ Can create and manage volumes  
✅ Understand bind mounts vs volumes vs tmpfs  
✅ Can configure volume drivers  
✅ Understand storage driver mechanics  
✅ Can setup NFS-backed volumes  
✅ Can backup/restore container data  
✅ Can design stateful apps (databases) in containers  
✅ Can troubleshoot storage issues
